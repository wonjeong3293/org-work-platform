# 개발 중인 시스템을 다른 사업장에 공유하는 방법

## 1. 현재 상황

개발 중인 업무 플랫폼(안전검사 대시보드 등)을 다른 사업장 부서원에게 보여주고 싶습니다.
현재는 개발자 PC에서만 접속할 수 있고, 다른 사업장에서는 접속이 안 됩니다.


## 2. 왜 안 되는지

우리 회사는 사업장마다 별도의 사내 네트워크(내부망)를 사용합니다.
쉽게 말하면, 화성 사업장과 평택 사업장이 서로 다른 건물에 있는 것처럼
네트워크도 서로 분리되어 있어서 직접 연결이 안 됩니다.

개발자 PC의 주소(예: 192.168.x.x)는 같은 사업장 안에서만 통하는 내부 주소입니다.
다른 사업장에서는 이 주소로 접속할 수 없습니다.


## 3. IT 부서에 요청할 내용

아래 내용을 IT 부서에 전달해 주세요.
메일이나 사내 메신저로 보내시면 됩니다.

---

### [IT 요청 내용]

제목: 개발 중인 내부 웹 시스템 타 사업장 임시 접속 요청

안녕하세요, OOO팀 OOO입니다.

현재 저희 팀에서 개발 중인 내부 업무 시스템(웹 기반)이 있습니다.
이 시스템을 다른 사업장 부서원에게 테스트 목적으로 공유하고 싶은데,
사업장 간 네트워크가 분리되어 있어 접속이 안 되는 상황입니다.

아래 방법 중 가능한 것이 있는지 확인 부탁드립니다.

방법 1) 양쪽 사업장에서 접속 가능한 사내 서버가 있다면 알려주세요.
        해당 서버에 시스템을 올려서 테스트하겠습니다.

방법 2) 개발자 PC의 특정 포트(3000번)를 다른 사업장에서
        접속할 수 있도록 임시 개방이 가능한지 확인 부탁드립니다.

방법 3) 위 방법이 어렵다면, 타 사업장 공유를 위해
        추천해주시는 다른 방법이 있을까요?

용도: 내부 업무 시스템 개발 테스트 (외부 공개 아님)
기간: 약 2주 (테스트 완료 후 종료)
접속 인원: 약 5명 이내

감사합니다.

---


## 4. IT에서 답변이 오면

IT에서 서버 주소나 방법을 알려주면, 저에게 전달해 주세요.
해당 환경에 맞게 시스템을 배포해 드리겠습니다.

만약 IT 승인까지 시간이 걸린다면,
당장은 Teams/Zoom 화면 공유로 시연하는 것도 방법입니다.
