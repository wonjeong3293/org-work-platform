# 프로토타입 개발 프롬프트 템플릿

## 사용법

1. Claude Code 웹사이트(claude.ai)에서 새 프로젝트 생성
2. `PROTOTYPE_GUIDE.md` 파일을 프로젝트 지식(Project Knowledge)에 추가
3. 아래 템플릿을 복사하여 첫 메시지로 입력

---

## 기본 템플릿

```
첨부한 PROTOTYPE_GUIDE.md는 우리 조직의 메인 시스템 개발 가이드야.
이 가이드의 기술 스택, 컴포넌트, 코딩 패턴을 반드시 따라서 개발해줘.

지금부터 [업무명] 프로토타입을 만들 거야.

## 기능 요구사항
- [기능 1]
- [기능 2]
- [기능 3]

## 필요한 페이지
- 목록 페이지: [설명]
- 등록/수정 페이지: [설명]
- 상세 페이지: [설명]

## 데이터 항목
- [필드1]: [타입] (예: 제목: 문자열, 필수)
- [필드2]: [타입]
- [필드3]: [타입]

사업장(siteCode)과 연도(year) 필터가 [필요함/필요없음].
```

## 작성 예시 1: 교육 관리

```
첨부한 PROTOTYPE_GUIDE.md는 우리 조직의 메인 시스템 개발 가이드야.
이 가이드의 기술 스택, 컴포넌트, 코딩 패턴을 반드시 따라서 개발해줘.

지금부터 교육 관리 프로토타입을 만들 거야.

## 기능 요구사항
- 교육 계획 등록/수정/삭제
- 교육 이수자 관리 (참석 체크)
- 교육 현황 대시보드 (월별 통계)
- 교육 이력 조회 및 엑셀 다운로드

## 필요한 페이지
- 목록 페이지: 교육 계획 목록을 테이블로 표시, 상태별 필터
- 등록 페이지: 교육 정보 입력 폼 (다이얼로그)
- 상세 페이지: 교육 상세 정보 + 이수자 명단 관리

## 데이터 항목
- 교육명: 문자열, 필수
- 교육일시: 날짜+시간, 필수
- 교육장소: 문자열
- 교육구분: 법정교육/자체교육/외부교육
- 교육시간: 숫자 (시간 단위)
- 강사명: 문자열
- 대상인원: 숫자
- 교육내용: 리치텍스트
- 첨부파일: 파일 업로드 (교육자료)

사업장(siteCode)과 연도(year) 필터가 필요함.
```

## 작성 예시 2: 자재 입출고 관리

```
첨부한 PROTOTYPE_GUIDE.md는 우리 조직의 메인 시스템 개발 가이드야.
이 가이드의 기술 스택, 컴포넌트, 코딩 패턴을 반드시 따라서 개발해줘.

지금부터 자재 입출고 관리 프로토타입을 만들 거야.

## 기능 요구사항
- 자재 마스터 등록/수정
- 입고/출고 처리
- 재고 현황 조회 (실시간 수량)
- 입출고 이력 조회

## 필요한 페이지
- 자재 목록: 자재 마스터 테이블 + 현재 재고량 표시
- 입출고 등록: 입고/출고 폼 (다이얼로그)
- 재고 현황: 카드형 대시보드 + 차트
- 이력 조회: 입출고 내역 테이블, 날짜 범위 필터

## 데이터 항목
자재 마스터:
- 자재코드: 문자열, 필수, 고유
- 자재명: 문자열, 필수
- 규격: 문자열
- 단위: EA/KG/M/SET
- 안전재고: 숫자
- 보관위치: 문자열

입출고:
- 자재: 자재 마스터 참조
- 구분: 입고/출고
- 수량: 숫자, 필수
- 일자: 날짜, 필수
- 비고: 문자열

사업장(siteCode) 필터가 필요함. 연도 필터는 필요없음.
```

## 작성 예시 3: 회의록 관리

```
첨부한 PROTOTYPE_GUIDE.md는 우리 조직의 메인 시스템 개발 가이드야.
이 가이드의 기술 스택, 컴포넌트, 코딩 패턴을 반드시 따라서 개발해줘.

지금부터 회의록 관리 프로토타입을 만들 거야.

## 기능 요구사항
- 회의록 작성 (리치텍스트 에디터)
- 참석자 선택
- 회의 결정사항 및 후속조치 관리
- 회의록 PDF 출력

## 필요한 페이지
- 목록 페이지: 회의록 카드 리스트, 검색/필터
- 작성 페이지: 회의 정보 + Tiptap 에디터로 내용 작성
- 상세 페이지: 회의록 뷰 + 후속조치 체크리스트

## 데이터 항목
- 회의명: 문자열, 필수
- 회의일시: 날짜+시간, 필수
- 회의장소: 문자열
- 회의구분: 정기회의/임시회의/프로젝트회의
- 참석자: 사용자 다중 선택
- 회의내용: 리치텍스트 (Tiptap)
- 결정사항: 리스트 (내용 + 담당자 + 기한)
- 첨부파일: 다중 파일

사업장(siteCode)과 연도(year) 필터가 필요없음.
```

## 추가 기능 요청 시

프로토타입 개발 중 추가 기능이 필요하면:

```
가이드 패턴에 맞춰서 [기능명]을 추가해줘.

- [요구사항 설명]
- [데이터 항목]
```

## 완료 후 전달할 것

프로토타입 완성 후 담당자에게 전달할 산출물:

1. **캡쳐 화면** — 주요 페이지 스크린샷
2. **코드 파일** — 아래 폴더의 파일들:
   - `src/app/(main)/[feature]/` — 페이지 파일
   - `src/components/[feature]/` — 컴포넌트 파일
   - `src/actions/[feature]-actions.ts` — 서버 액션
3. **Prisma 모델 정의** — schema.prisma에 추가한 모델 부분
4. **상수 정의** — constants.ts에 추가한 상수 부분
5. **간단한 기능 설명 문서** — 어떤 기능이 있는지 요약
